# CardListExpandable

O componente CardListExpandable é um card interativo que permite expandir e colapsar conteúdo adicional. Ideal para listas de transações, detalhes de pedidos, informações adicionais ou qualquer conteúdo que precise ser ocultado inicialmente.

## Importação

```tsx
import { CardListExpandable } from '@useblu/ocean-react';
```

### Importação específica (recomendado para tree-shaking)

```tsx
import { CardListExpandable } from '@useblu/ocean-react/CardListExpandable';
```

### Importação de tipos TypeScript

```tsx
import type { CardListExpandableProps } from '@useblu/ocean-react';

// Uso em componente customizado
const MyComponent: React.FC<CardListExpandableProps> = (props) => {
  return <CardListExpandable {...props} />;
};
```

### Uncontrolled (padrão)

```tsx
// O componente gerencia seu próprio estado
<CardListExpandable
  title="Card Uncontrolled"
  description="Gerencia o estado internamente"
  defaultExpanded={false}
  onToggle={(expanded) => console.log('Estado:', expanded)}
>
  <div>Conteúdo expandido</div>
</CardListExpandable>
```

### Controlled

```tsx
// Você controla o estado externamente
const [expanded, setExpanded] = useState(false);

<CardListExpandable
  title="Card Controlled"
  description="Controlado por estado externo"
  expanded={expanded}
  onToggle={(newExpanded) => setExpanded(newExpanded)}
>
  <div>Conteúdo expandido</div>
</CardListExpandable>
```

### Guia de uso dos tipos

- **default**: Tipo padrão para conteúdo geral
- **inactive**: Para itens inativos ou desabilitados
- **positive**: Para valores positivos (créditos, ganhos)
- **warning**: Para itens que precisam de atenção
- **highlight**: Para informações importantes
- **highlight-lead**: Para destaques principais com mais ênfase
- **strikethrough**: Para valores com desconto (mostra valor riscado)

## API

### Props

| Prop | Tipo | Padrão | Descrição |
| ---- | ---- | ------ | --------- |
| `title` | `string` | - | Título principal do card (obrigatório) |
| `description` | `string` | - | Descrição ou texto secundário |
| `strikethroughDescription` | `string` | - | Texto riscado (usado com type="strikethrough") |
| `caption` | `string` | - | Legenda ou texto terciário |
| `inverted` | `boolean` | `false` | Inverte posição título/descrição |
| `type` | `'default' \| 'inactive' \| 'positive' \| 'warning' \| 'highlight' \| 'highlight-lead' \| 'strikethrough'` | `'default'` | Tipo de estilo visual |
| `loading` | `boolean` | `false` | Mostra skeleton de carregamento |
| `icon` | `ReactNode` | - | Ícone à esquerda do conteúdo |
| `indicator` | `ReactNode` | - | Badge/Tag antes da ação de expansão |
| `expanded` | `boolean` | - | Controla estado expandido (controlled) |
| `defaultExpanded` | `boolean` | `false` | Estado inicial (uncontrolled) |
| `onToggle` | `(expanded: boolean) => void` | - | Callback ao expandir/colapsar |
| `children` | `ReactNode` | - | Conteúdo exibido quando expandido |
| `className` | `string` | - | Classes CSS adicionais |

### Eventos

- **onToggle**: Disparado quando o card é expandido ou colapsado, recebe o novo estado como parâmetro

## Acessibilidade

- ✅ Elemento `<button>` semântico para ação de expansão
- ✅ Suporte completo a navegação por teclado
- ✅ Estados de foco visíveis
- ✅ Indicadores visuais claros (ícones de chevron)
- ✅ Transições suaves entre estados
- ✅ Skeleton acessível durante loading

### Navegação por teclado

| Tecla | Função |
| ----- | ------ |
| `Tab` | Move foco para o card |
| `Space` / `Enter` | Expande ou colapsa o conteúdo |
| `Shift + Tab` | Move foco para elemento anterior |

## Melhores práticas

### ✅ Faça

- Use títulos descritivos que indiquem o conteúdo
- Adicione indicadores (badges/tags) para informações importantes
- Use `type` apropriado para comunicar significado (positive, warning, etc.)
- Combine com `List` para múltiplos cards
- Use estado `loading` durante carregamento de dados
- Implemente `onToggle` para tracking de interações
- Use modo `inverted` para destacar valores monetários
- Forneça ícones significativos que identifiquem o tipo de conteúdo

### ❌ Não faça

- Não oculte informações críticas no conteúdo expandido
- Não use conteúdo expandido excessivamente longo (considere split)
- Não omita o título (é obrigatório)
- Não ignore o estado `loading` durante carregamentos
- Não abuse de cards expandidos na mesma tela (pode confundir)
- Não use múltiplos níveis de expansão (evite nesting)
