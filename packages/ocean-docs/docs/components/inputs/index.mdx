# Input

Os componentes de Input são fundamentais para coletar dados do usuário em formulários e interfaces. O Ocean Design System oferece uma variedade de tipos de input otimizados para diferentes tipos de dados e contextos de uso.

## Importação

```tsx
import { Input } from '@useblu/ocean-react';
```

### Importação específica (recomendado para tree-shaking)

```tsx
import { Input } from '@useblu/ocean-react/Input';
```

### Importação de tipos TypeScript

```tsx
import type { InputProps } from '@useblu/ocean-react';

// Uso em componente customizado
const MyInput: React.FC<InputProps> = (props) => {
  return <Input {...props} />;
};
```

### Text Input

Componente básico para entrada de texto livre, ideal para nomes, descrições e dados textuais gerais.

**Use quando:**

- Coletar nomes, títulos ou descrições
- Entrada de texto livre sem formatação específica
- Dados textuais que não requerem validação especial

### Email Input

Especializado para entrada de endereços de email com validação integrada.

**Use quando:**

- Coletar endereços de email
- Formulários de login/cadastro
- Campos que requerem validação de formato de email

### Password Input

Otimizado para entrada de senhas com recursos de segurança.

**Use quando:**

- Coletar senhas
- Confirmação de senhas
- Campos que requerem entrada confidencial

### Number Input

Projetado para entrada numérica com controles específicos.

**Use quando:**

- Coletar números inteiros ou decimais
- Quantidades, preços ou valores numéricos
- Campos que requerem validação numérica

### Currency Input

Especializado para entrada de valores monetários com formatação automática.

**Use quando:**

- Coletar valores em dinheiro
- Preços, salários ou valores financeiros
- Campos que requerem formatação de moeda

## Características Comuns

Todos os componentes de Input compartilham características essenciais:

### ✅ Acessibilidade

- Navegação por teclado completa
- Suporte a screen readers
- Estados de foco visíveis
- Atributos ARIA apropriados

### ✅ Estados Visuais

- Estado normal para entrada padrão
- Estado de erro para validação
- Estado desabilitado quando não editável
- Estado de foco para interação ativa

### ✅ Validação

- Suporte a validação em tempo real
- Mensagens de erro contextuais
- Indicadores visuais de estado
- Integração com bibliotecas de formulário

### ✅ Customização

- Estilos consistentes com o design system
- Props flexíveis para diferentes contextos
- Suporte a temas e variações
- Classes CSS customizáveis

## Estrutura Comum

Todos os inputs seguem uma estrutura consistente:

```tsx
<Input
  label="Rótulo do Campo" // Identificação clara do propósito
  name="fieldName" // Nome para formulários
  type="text" // Tipo específico do input
  placeholder="Exemplo de entrada" // Orientação visual
  helperText="Texto de ajuda" // Contexto adicional
  required // Indicação de obrigatoriedade
  error={hasError} // Estado de validação
  disabled={isDisabled} // Estado de interação
/>
```

## Melhores Práticas Gerais

### ✅ Labels e Orientação

- Sempre forneça labels claros e descritivos
- Use helperText para orientações adicionais
- Placeholders devem exemplificar o formato esperado
- Marque campos obrigatórios adequadamente

### ✅ Validação e Feedback

- Forneça feedback imediato quando apropriado
- Use mensagens de erro específicas e úteis
- Indique claramente o estado de validação
- Mantenha consistência nas mensagens

### ✅ Agrupamento e Layout

- Agrupe campos relacionados visualmente
- Use ordem lógica nos formulários
- Mantenha espaçamento consistente
- Considere o fluxo de navegação

### ❌ Evite

- Labels ambíguos ou técnicos demais
- Mensagens de erro genéricas
- Campos desabilitados sem explicação
- Validação excessivamente rigorosa durante digitação
