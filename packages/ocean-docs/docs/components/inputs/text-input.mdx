# Text Input

Componente especializado de entrada de texto para coletar dados textuais gerais com recursos de validação e acessibilidade.

## Importação

```tsx
import { Input } from '@useblu/ocean-react';
```

### Importação específica (recomendado para tree-shaking)

```tsx
import { Input } from '@useblu/ocean-react/Input';
```

### Importação de tipos TypeScript

```tsx
import type { InputProps } from '@useblu/ocean-react';

// Uso em componente customizado
const MyTextInput: React.FC<InputProps> = (props) => {
  return <Input {...props} type="text" />;
};
```

## Estados do Input

O Text Input suporta diferentes estados para diferentes situações:

### Estado Normal

```tsx live
<Input
  label="Nome"
  name="name"
  type="text"
  placeholder="João Silva"
  helperText="Nome que aparecerá no perfil"
/>
```

### Estado de Erro

```tsx live
<Input
  label="Email"
  name="email"
  type="text"
  placeholder="seu@email.com"
  helperText="Campo obrigatório"
  error
/>
```

### Estado Desabilitado

```tsx live
<Input
  label="Nome"
  name="name"
  type="text"
  placeholder="João Silva"
  helperText="Campo desabilitado"
  disabled
/>
```

### Campo Obrigatório

```tsx live
<Input
  label="Nome Completo"
  name="fullName"
  type="text"
  placeholder="João Silva"
  helperText="Primeiro e último nome"
  required
/>
```

## API

### Props

| Prop           | Tipo                           | Padrão   | Descrição                                              |
| -------------- | ------------------------------ | -------- | ------------------------------------------------------ |
| `label`        | `string`                       | -        | O rótulo do campo que identifica seu propósito         |
| `name`         | `string`                       | -        | Nome do campo para identificação no formulário         |
| `type`         | `"text"`                       | `"text"` | Tipo do input, sempre "text" para entrada de texto     |
| `placeholder`  | `string`                       | -        | Texto de exemplo que aparece quando o campo está vazio |
| `helperText`   | `string`                       | -        | Texto auxiliar que fornece contexto ou instruções      |
| `error`        | `boolean`                      | `false`  | Define se o campo está em estado de erro               |
| `disabled`     | `boolean`                      | `false`  | Desabilita a interação com o campo                     |
| `required`     | `boolean`                      | `false`  | Marca o campo como obrigatório                         |
| `defaultValue` | `string`                       | -        | Valor padrão do campo                                  |
| `className`    | `string`                       | -        | Classes CSS adicionais                                 |
| `onChange`     | `(event: ChangeEvent) => void` | -        | Função chamada quando o valor muda                     |
| `onFocus`      | `(event: FocusEvent) => void`  | -        | Função chamada quando o campo recebe foco              |
| `onBlur`       | `(event: FocusEvent) => void`  | -        | Função chamada quando o campo perde foco               |

- ✅ Suporte completo a navegação por teclado
- ✅ Estados de foco visíveis

- ✅ Suporte a screen readers
- ✅ Atributos ARIA apropriados
- ✅ Mensagens de erro acessíveis

### Navegação por teclado

| Tecla       | Função                                   |
| ----------- | ---------------------------------------- |
| `Tab`       | Move o foco para o próximo elemento      |
| `Shift+Tab` | Move o foco para o elemento anterior     |
| `Enter`     | Submete o formulário (se dentro de form) |

## Melhores práticas

### ✅ Faça

- Use labels claros e descritivos que expliquem o propósito do campo
- Forneça texto de ajuda (`helperText`) quando necessário para orientar o usuário
- Use placeholders que exemplifiquem o formato esperado dos dados
- Marque campos obrigatórios com a prop `required`
- Use estado de erro (`error`) com mensagens informativas no `helperText`
- Agrupe campos relacionados visualmente (nome/sobrenome, endereço completo)

### ❌ Não faça

- Não use apenas placeholder como label, sempre forneça um `label` adequado
- Não deixe mensagens de erro genéricas, seja específico sobre o problema
- Não desabilite campos sem uma explicação clara no `helperText`
- Não omita validação em campos obrigatórios
- Não use placeholders muito longos que possam confundir o usuário

## Validação e Estados

### Exemplo com validação

```tsx
const [name, setName] = useState('');
const [error, setError] = useState(false);

const handleChange = (e) => {
  const value = e.target.value;
  setName(value);
  setError(value.length < 2);
};

return (
  <Input
    label="Nome Completo"
    name="fullName"
    type="text"
    value={name}
    onChange={handleChange}
    error={error}
    helperText={
      error ? 'Nome deve ter pelo menos 2 caracteres' : 'Primeiro e último nome'
    }
    required
  />
);
```
