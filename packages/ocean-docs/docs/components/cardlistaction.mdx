# CardListAction

O componente CardListAction é um item de lista interativo que combina conteúdo textual com ações do usuário. Ideal para listas de configurações, menus de opções, navegação entre telas e exibição de informações acionáveis.

## Importação

```tsx
import { CardListAction } from '@useblu/ocean-react';
```

### Importação específica (recomendado para tree-shaking)

```tsx
import { CardListAction } from '@useblu/ocean-react/CardListAction';
```

### Importação de tipos TypeScript

```tsx
import type { CardListActionProps } from '@useblu/ocean-react';

// Uso em componente customizado
const MyCardListAction: React.FC<CardListActionProps> = (props) => {
  return <CardListAction {...props} />;
};
```

## Uso básico

```tsx live
<CardListAction
  title="Título do Card"
  description="Descrição ou informação adicional"

/>
```

## Tipos de conteúdo

O CardListAction oferece diferentes tipos visuais para destacar estados ou categorias de informação:

### Default

```tsx live
<CardListAction
  type="default"
  title="Configurações gerais"
  description="Gerencie suas preferências"
/>
```

### Inactive

```tsx live
<CardListAction
  type="inactive"
  title="Serviço indisponível"
  description="Este serviço está temporariamente desabilitado"
/>
```

### Positive

```tsx live
<CardListAction
  type="positive"
  title="Pagamento confirmado"
  description="R$ 150,00"
/>
```

### Warning

```tsx live
<CardListAction
  type="warning"
  title="Atenção necessária"
  description="Verifique suas informações"
/>
```

### Highlight

```tsx live
<CardListAction
  type="highlight"
  title="Novidade!"
  description="Conheça nossa nova funcionalidade"
/>
```

### Highlight Lead

```tsx live
<CardListAction
  type="highlight-lead"
  title="R$ 1.250,00"
  description="Valor destacado"
/>
```

### Strikethrough

```tsx live
<CardListAction
  type="strikethrough"
  title="Oferta expirada"
  description="Esta promoção já encerrou"
/>
```

### Guia de uso dos tipos

- **default**: Tipo padrão para itens comuns de lista
- **inactive**: Para itens desabilitados ou indisponíveis
- **positive**: Para destacar informações positivas (sucesso, confirmação)
- **warning**: Para alertas ou itens que necessitam atenção
- **highlight**: Para destacar novidades ou itens importantes
- **highlight-lead**: Para destacar valores monetários ou métricas principais
- **strikethrough**: Para itens obsoletos ou descontinuados

## Conteúdo invertido

O prop `inverted` inverte a posição do título e da descrição, útil para priorizar valores ou descrições:

```tsx live
<div style={{ display: 'flex', flexDirection: 'column', gap: '16px' }}>
  <CardListAction title="Saldo disponível" description="R$ 1.250,00" />

  <CardListAction inverted title="R$ 1.250,00" description="Saldo disponível" />
</div>
```

## Descrição com tachado

Use `strikethroughDescription` para mostrar valores antigos ou informações obsoletas:

```tsx live
<CardListAction
  title="Produto em promoção"
  strikethroughDescription="De R$ 299,00"
  description="Por R$ 199,00"
/>
```

## Caption (legenda)

Adicione uma terceira linha de texto com o prop `caption`:

```tsx live
<CardListAction
  title="Transferência realizada"
  description="R$ 500,00"
  caption="Hoje às 14:32"
/>
```

## Ícones

Adicione ícones à esquerda para melhor identificação visual:

```tsx live
<CardListAction
  icon={<PlaceholderOutline size={24} />}
  title="Meus documentos"
  description="Acesse seus arquivos"
/>
```

## Indicadores

Use indicadores (badges, tags) à direita do conteúdo antes da ação:

```tsx live
<div style={{ display: 'flex', flexDirection: 'column', gap: '16px' }}>
  <CardListAction
    title="Notificações"
    description="Você tem novas mensagens"

  />

  <CardListAction
    title="Novo recurso"
    description="Experimente agora"
    indicator={<Tag label="Novo" color="highlight" />}
  />
</div>
```

## Tipos de ação

O CardListAction suporta três tipos de ação: chevron (padrão), menu e swipe.

### Chevron (navegação)

Tipo padrão que indica navegação para outra tela:

```tsx live
<CardListAction
  actionType="chevron"
  title="Minha conta"
  description="Gerenciar informações pessoais"
  onClick={() => alert('Navegar para Minha conta')}
/>
```

### Menu (ações múltiplas)

Exibe um menu dropdown com múltiplas ações:

```tsx live
<CardListAction
  actionType="menu"
  title="Documento importante"
  description="relatório-anual.pdf"
  menuActions={[
    {
      label: 'Editar',
      icon: <Pencil size={20} />,
      onClick: () => alert('Editar'),
    },
    {
      label: 'Compartilhar',
      icon: <Share size={20} />,
      onClick: () => alert('Compartilhar'),
    },
    {
      label: 'Arquivar',
      icon: <Archive size={20} />,
      onClick: () => alert('Arquivar'),
    },
    {
      label: 'Excluir',
      icon: <Trash size={20} />,
      onClick: () => alert('Excluir'),
      variant: 'critical',
    },
  ]}
/>
```

### Swipe (ações rápidas)

Permite revelar ações deslizando o item (mobile-friendly):

```tsx live
<CardListAction
  actionType="swipe"
  title="Email importante"
  description="Arraste para ver ações"
  menuActions={[
    {
      label: 'Editar',
      icon: <Pencil size={20} />,
      onClick: () => alert('Editar'),
    },
    {
      label: 'Arquivar',
      icon: <Archive size={20} />,
      onClick: () => alert('Arquivar'),
    },
    {
      label: 'Excluir',
      icon: <Trash size={20} />,
      onClick: () => alert('Excluir'),
      variant: 'critical',
    },
  ]}
/>
```

### Posicionamento do menu

Para o tipo `menu`, você pode controlar a posição do dropdown:

```tsx live
<div style={{ display: 'flex', flexDirection: 'column', gap: '16px' }}>
  <CardListAction
    actionType="menu"
    menuPosition="bottom-right"
    title="Menu à direita (padrão)"
    description="Abrir dropdown"
    menuActions={[
      {
        label: 'Ação 1',
        onClick: () => alert('Ação 1'),
      },
    ]}
  />

  <CardListAction
    actionType="menu"
    menuPosition="bottom-left"
    title="Menu à esquerda"
    description="Abrir dropdown"
    menuActions={[
      {
        label: 'Ação 1',
        onClick: () => alert('Ação 1'),
      },
    ]}
  />
</div>
```

## API

### Props

| Prop                       | Tipo                                                                                                       | Padrão           | Descrição                                  |
| -------------------------- | ---------------------------------------------------------------------------------------------------------- | ---------------- | ------------------------------------------ |
| `title`                    | `string`                                                                                                   | -                | Título principal do card (obrigatório)     |
| `description`              | `string`                                                                                                   | -                | Descrição ou texto secundário              |
| `strikethroughDescription` | `string`                                                                                                   | -                | Descrição com texto riscado (valor antigo) |
| `caption`                  | `string`                                                                                                   | -                | Legenda ou texto terciário                 |
| `inverted`                 | `boolean`                                                                                                  | `false`          | Inverte posição do título com descrição    |
| `type`                     | `'default' \| 'inactive' \| 'positive' \| 'warning' \| 'highlight' \| 'highlight-lead' \| 'strikethrough'` | `'default'`      | Tipo de estilo visual do conteúdo          |
| `disabled`                 | `boolean`                                                                                                  | `false`          | Desabilita o card                          |
| `loading`                  | `boolean`                                                                                                  | `false`          | Exibe skeleton de carregamento             |
| `icon`                     | `ReactNode`                                                                                                | -                | Ícone exibido no início do card            |
| `indicator`                | `ReactNode`                                                                                                | -                | Indicador (badge/tag) antes da ação        |
| `actionType`               | `'chevron' \| 'menu' \| 'swipe'`                                                                           | `'chevron'`      | Tipo de ação do card                       |
| `menuActions`              | `ActionItem[]`                                                                                             | -                | Lista de ações para tipos menu/swipe       |
| `menuPosition`             | `'bottom-left' \| 'bottom-right' \| 'top-left' \| 'top-right'`                                             | `'bottom-right'` | Posição do menu dropdown                   |
| `onClick`                  | `() => void`                                                                                               | -                | Função chamada ao clicar no card           |
| `className`                | `string`                                                                                                   | -                | Classes CSS adicionais                     |

### ActionItem (menuActions)

| Prop      | Tipo                      | Descrição                              |
| --------- | ------------------------- | -------------------------------------- |
| `label`   | `string`                  | Texto da ação                          |
| `icon`    | `ReactNode`               | Ícone da ação                          |
| `onClick` | `() => void`              | Função executada ao clicar na ação     |
| `variant` | `'default' \| 'critical'` | Variante visual (crítica para deletar) |

### Eventos

- `onClick`: Disparado quando o card é clicado (não aplicável para tipo swipe)
- Cada `ActionItem` tem seu próprio `onClick` independente

## Acessibilidade

- ✅ Elemento `<button>` semântico para acessibilidade
- ✅ Suporte completo a navegação por teclado
- ✅ Estados de foco visíveis
- ✅ Estados disabled adequadamente sinalizados
- ✅ Skeleton acessível durante loading
- ✅ Menu dropdown acessível com navegação por teclado
- ✅ Ações de swipe com alternativa por teclado

### Navegação por teclado

| Tecla             | Função                                  |
| ----------------- | --------------------------------------- |
| `Tab`             | Move o foco para o card ou menu         |
| `Space` / `Enter` | Ativa o card ou abre o menu             |
| `Escape`          | Fecha o menu dropdown (tipo menu)       |
| `↑` / `↓`         | Navega entre opções do menu (tipo menu) |

## Melhores práticas

### ✅ Faça

- Use títulos claros e descritivos que indiquem o conteúdo ou ação
- Combine com List para agrupar múltiplos cards relacionados
- Use o tipo apropriado para comunicar estados (positive, warning, etc.)
- Utilize indicadores (badges/tags) para informações de status
- Prefira `actionType="chevron"` para navegação simples
- Use `actionType="menu"` quando houver múltiplas ações disponíveis
- Use `actionType="swipe"` em interfaces mobile para economia de espaço
- Forneça feedback visual durante operações com `loading`
- Use `inverted` quando o valor/descrição for mais importante que o título
- Limite menuActions a 3-5 itens para melhor usabilidade

### ❌ Não faça

- Não use textos muito longos que truncam em dispositivos pequenos
- Não omita o `onClick` quando o card for clicável
- Não use múltiplos indicadores no mesmo card (mantém o design limpo)
- Não misture tipos de ação inconsistentemente na mesma lista
- Não use tipo `strikethrough` sem contexto claro
- Não coloque ações críticas (delete) sem confirmação
- Não sobrecarregue o menu com muitas opções
- Não use `disabled` sem indicar o motivo ao usuário
- Não ignore estados de loading em operações assíncronas
