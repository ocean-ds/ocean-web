## Overview <a name="overview"></a>

A badge is an essential component within a design system, designed to provide a visually
appealing way to highlight important information or draw the user's attention to specific
events, notifications, or statuses. It is often used in user interfaces to indicate the number
of unread notifications, the count of items in a shopping cart, the rating of an item, or
any other relevant data that needs to be emphasized.

## Single digits <a name="variants"></a>

Displays only one digit and its size is fixed.

## Multiple digits

Displays 2 or more digits and its size increases with the number of characters.
When there is an overflow above 99, a "+" is added to make the sizes default.

<Canvas>
  <Story name="Multiple digits">
    <ContentRow direction="row">
      <Badge count={10} />
      <Badge count={55} />
      <Badge count={100} />
    </ContentRow>
  </Story>
</Canvas>

## Dot

Informs that there is something new or needing to be resolved in a certain part of the interface.
It does not display any numerical value and can be used when there is no space for any other Badge.

<Canvas>
  <Story name="Dot">
    <ContentRow>
      <Badge variation="tiny" />
    </ContentRow>
  </Story>
</Canvas>

## Color

The default badge color is `$color-brand-primary-pure` but it is possible to use other colors depending
on the context. Colors should vary according to predefined options.

      <Badge variation="small" count={15} color="complementary" />

</Canvas>

## Sizes

The `variation` parameter of the Badge component allows for customization of the badge's dimensions,
ensuring it aligns harmoniously with the overall user interface. This parameter offers flexibility in tailoring the
badge's visual appearance based on the specific context and requirements.
