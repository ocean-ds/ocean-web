# Text Input

Specialized text input component for collecting general text data with validation and accessibility features.

## Overview

Text inputs are the most common form control for collecting user-entered text data. They support various validation states, helper text, and accessibility features. Use text inputs for names, addresses, descriptions, and other free-form text content.

## Import

```javascript
import { Input } from '@useblu/ocean-react';
```

## Usage

Text inputs are designed for collecting textual information from users with proper validation and feedback.

## API Reference

Text input specific properties and usage guidelines.

| Prop        | Type    | Default   | Description                                                                   |
| ----------- | ------- | --------- | ----------------------------------------------------------------------------- |
| type        | "text"  | "text"    | Always "text" for text inputs. Provides standard text input behavior.         |
| label       | string  | undefined | Label text for the input. Essential for accessibility and user understanding. |
| placeholder | string  | undefined | Example text shown when input is empty. Should demonstrate expected format.   |
| helperText  | string  | undefined | Additional guidance or validation messages for the user.                      |
| error       | boolean | false     | When true, displays error styling. Use with form validation.                  |
| disabled    | boolean | false     | When true, prevents user interaction. Use when field is not available.        |

## CSS

| Global class         | Description                                  |
| -------------------- | -------------------------------------------- |
| .ods-input           | Styles applied to the root element.          |
| .ods-input--filled   | Styles applied when input has content.       |
| .ods-input--disabled | Styles applied when input is disabled.       |
| .ods-input--error    | Styles applied when input is in error state. |
