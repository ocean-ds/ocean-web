# Sistema de Grid

O sistema de grid do Ocean fornece uma estrutura flexível e responsiva para organizar conteúdo em layouts consistentes e adaptáveis a diferentes tamanhos de tela.

## Visão Geral

### Estrutura Base

O grid do Ocean é baseado em **12 colunas** com breakpoints responsivos que se adaptam automaticamente a diferentes dispositivos.

### Componentes do Grid

- **Container**: Envolve o conteúdo e define largura máxima
- **Row**: Agrupa colunas horizontalmente
- **Col**: Define colunas individuais com larguras responsivas

## Breakpoints Responsivos

### Tamanhos de Tela

O sistema inclui breakpoints para diferentes dispositivos:

- **xs** (extra small): < 576px (celulares)
- **sm** (small): ≥ 576px (celulares grandes)
- **md** (medium): ≥ 768px (tablets)
- **lg** (large): ≥ 992px (desktops)
- **xl** (extra large): ≥ 1200px (desktops grandes)

## Componentes do Sistema

### Container

O `Container` centraliza e limita a largura máxima do conteúdo:

```jsx
<Grid.Container>{/* Seu conteúdo aqui */}</Grid.Container>
```

### Row

A `Row` cria agrupamentos horizontais de colunas:

```jsx
<Grid.Row>
  <Grid.Col>Coluna 1</Grid.Col>
  <Grid.Col>Coluna 2</Grid.Col>
</Grid.Row>
```

### Col

As `Col` definem larguras responsivas:

```jsx
// Exemplos de uso
<Grid.Col xs="12" md="6" lg="4">Responsiva</Grid.Col>
<Grid.Col md="8">8 colunas no desktop</Grid.Col>
<Grid.Col>Largura automática</Grid.Col>
```

## API dos Componentes

### Grid.Container

```jsx
<Grid.Container>{children}</Grid.Container>
```

**Props:**

- `children`: Conteúdo do container

### Grid.Row

```jsx
<Grid.Row>{children}</Grid.Row>
```

**Props:**

- `children`: Colunas filhas

### Grid.Col

```jsx
<Grid.Col xs="12" sm="6" md="4" lg="3" xl="2">
  {children}
</Grid.Col>
```

**Props:**

- `xs`: Largura em telas extra pequenas (< 576px)
- `sm`: Largura em telas pequenas (≥ 576px)
- `md`: Largura em telas médias (≥ 768px)
- `lg`: Largura em telas grandes (≥ 992px)
- `xl`: Largura em telas extra grandes (≥ 1200px)
- `children`: Conteúdo da coluna

**Valores aceitos:** "1" a "6" (representando colunas de 1 a 12)

## Melhores Práticas

### ✅ Faça

- Use `Container` para envolver layouts principais
- Agrupe colunas relacionadas em `Row`
- Defina larguras para diferentes breakpoints
- Teste layouts em diferentes tamanhos de tela
- Use grid aninhado para layouts complexos

### ❌ Evite

- Colocar conteúdo diretamente no `Container` sem `Row`
- Usar larguras que não somem 12 colunas
- Criar layouts muito complexos com muitos níveis aninhados
- Ignorar comportamento mobile
- Usar grid para alinhamentos simples (prefira flexbox)

## Exemplos de Uso

### Layout de Cards

```jsx
<Grid.Container>
  <Grid.Row>
    {cards.map((card) => (
      <Grid.Col xs="12" sm="6" lg="4" key={card.id}>
        <Card>{card.content}</Card>
      </Grid.Col>
    ))}
  </Grid.Row>
</Grid.Container>
```

### Layout de Formulário

```jsx
<Grid.Container>
  <Grid.Row>
    <Grid.Col xs="12" md="6">
      <Input label="Nome" />
    </Grid.Col>
    <Grid.Col xs="12" md="6">
      <Input label="Sobrenome" />
    </Grid.Col>
    <Grid.Col xs="12">
```
